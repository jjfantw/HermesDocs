專案四：內部作業系統（Internal Operating System）

9:52

Austin 認為這個專案讓他從「AI 很厲害」進化到「AI 會永遠改變世界」。前三者是工具，而內部作業系統是將所有工具整合在一起的基礎架構。

第一步：建立結構。 三個核心資料夾：(1) Knowledge — 你想讓 Claude 知道的一切：會議記錄、語音樣本、你信任的框架、收藏的文章、顧問團的內容；(2) Skills — 可重複使用的流程，如專案一的 ask the board 技能；(3) Project — 你正在進行的專案，如指令中心和個人網站。根目錄的 `CLAUDE.md` 是整個系統的大腦，告訴 Claude 如何使用這些資料夾。

第二步：建立 /improve-system 技能。 這是整個流程最重要的部分。多數人拿到 AI 的輸出後就接受了。關鍵在於建立一個能記錄資訊（第一步）並捕捉回饋的系統。例如你寫了一封太冗長的 email，說「rewrite shorter」，定稿後執行 `/improve-system`，下次 AI 寫 email 就會自動更簡潔。每次迭代都在教導系統並持續進步。

第三步：建立 ingest-resource 技能。 improve-system 是老師，ingest-resource 是圖書館員。它能系統性地將新資訊（文章、逐字稿、YouTube 影片）攝入系統，確保 AI 知道去哪裡找。

第四步：上傳 GitHub。 雖然非強制，但 Austin 認為內部作業系統是你人生的智慧財產。放在 GitHub 上可以輕鬆復原變更、追蹤進度，也讓團隊成員能接入。

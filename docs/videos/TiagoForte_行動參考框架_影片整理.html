章節整理

00:00 資訊過載 — 為什麼找不到東西
訊息、會議、通知、文件、書、播客、YouTube — 全部都是資訊。你知道「明明存過」但找不到,那是因為你試著用石器時代的大腦處理 21 世紀的資訊量。該讓 App 接手一部份負載。
00:50 兩個腦半球 — 行動 vs 參考
就像大腦的左右半球,你的「第二大腦」也可以分兩邊:左 = 分析 / 行動(任務、待辦、會議);右 = 自由 / 參考(學習、思考、未來可能用到的內容)。這套分流最早來自 David Allen《Getting Things Done》。
01:38 行動側 — 行事曆 vs 待辦
行動側再切一刀:有指定時間的(會議、Zoom、活動)→ 行事曆;用 Google Calendar 因為穩、可分享;沒指定時間的(寫文章)→ 待辦清單;新手用 Apple Reminders,進階用 Things 3。重點是:不要把待辦塞進參考側的雜訊裡。
03:42 參考側 — 筆記 vs 雲端
筆記 App 是你的工作室/實驗室 — 開工第一件事就是打開它;適合書中一句話、想法、連結、照片。但 PDF、共享文件、太大或需保留原格式的東西,放雲端硬碟(Google Drive)。書中引文 → 筆記;研究用的 PDF → Drive。
05:10 結論 — It's not you, it's your system
把生產力想成「系統」而不是「你自己」。當資訊過載感升起,不要怪自己,問:「我系統的哪裡出錯了?」這就是 PARA 的出場點 — 它橫切四個象限,給你一套組織所有資訊的統一規則(本片未展開 PARA,留到後續影片)。

關鍵洞察

  • 「第二大腦」是系統,不是 App。同一支 App(例如 Notion)可以裝四象限的內容;框架決定你「怎麼放」,不是叫你「換工具」。
  • Yes/No 二分法比資料夾分類省腦。不必想十層資料夾,只要問兩個問題就能決定歸位。
  • 把行動從參考的雜訊中拉出來。待辦淹沒在有趣的內容裡,自然會忘;放進獨立 App 才會被完成。
  • 筆記 App = 主戰場。作者每天開工第一件事就是打開它 — 它是工作室,不是垃圾桶。
  • 不是你的問題,是系統的問題。當你感到資訊過載,先檢查系統,不要自責。
  • PARA 是這支影片的延伸鉤子。本片只點到「橫切四象限」,沒展開 Projects / Areas / Resources / Archives — 那是 Forte Labs 下一支影片。

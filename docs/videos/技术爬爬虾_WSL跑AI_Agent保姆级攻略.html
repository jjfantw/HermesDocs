為什麼 WSL 是 Windows 跑 AI Agent 的最佳解

00:00

AI Agent 在 Windows 上的表現不如 MacOS，主要原因就是 PowerShell 對 AI 不夠友好。AI 大模型訓練語料中的命令列操作大部分是 Linux 和 MacOS 風格，在 Linux 環境中出錯率明顯更低、效果更好。

WSL（Windows Subsystem for Linux）只需一行命令就能在 Windows 上運行 Linux 子系統，讓電腦同時獲得兩大作業系統的優勢。在 WSL 中運行 AI Agent 可獲得三大好處：Agent 運行更穩效率更高（Linux 命令出錯率低、省 Token）、與生產環境更一致（程式最終部署到 Linux 伺服器，環境差異小）、環境隔離更安全（WSL 提供 Linux 虛擬機與 Windows 宿主機之間的隔離，搞壞了刪掉重建即可）。

WSL 安裝與基礎使用

01:33

安裝前需確認 CPU 虛擬化已啟用：在任務管理器的「效能 > CPU」中查看「虛擬化」狀態。若未啟用，需進入 BIOS 開啟 Intel VMX 或 AMD SVM Mode。

在 Windows Terminal 中輸入 `wsl --install`（國內網路建議加 `--web-download` 提高下載速度），預設安裝 Ubuntu。安裝過程中需設定 Linux 使用者名稱和密碼（輸入密碼時螢幕不會顯示內容）。安裝完成後命令列顏色改變，即進入 Linux 子系統，輸入 `exit` 可退出。

可安裝其他發行版：`wsl --list --online` 列出所有可安裝版本，`wsl --install <發行版名>` 指定安裝，`--location` 可指定安裝目錄。同一發行版可建立多個隔離實例（`wsl --install Ubuntu --name Ubuntu2`）。`wsl --list` 列出所有已安裝的 Linux 虛擬機。

在 WSL 中運行 AI Agent（Py）

06:19

運行 Agent 前需確認開發三件套：Git（Ubuntu 自帶）、Python 3（自帶，需將 `python` 命令連結到 `python3`）、Node.js（透過 NVM 安裝版本 24）。

以 Py（極簡 Coding Agent）為例：使用 curl 安裝後，執行 `py --login` 配置 API Key（支援 DeepSeek、Minimax、Kimi 等），選擇模型後即可開始開發。

實戰示範：用 Next.js 框架編寫坦克大戰遊戲，開發完成後可直接用 Windows 上的瀏覽器（Edge）訪問 WSL 中的服務。這是因為 WSL 會自動將 Linux 中的端口轉發到 Windows 的 localhost——WSL 檢測到 Linux 中有程式監聽端口，就會自動轉發到 Windows 宿主機的 localhost 相同端口。

VSCode 整合與 Git 操作

08:48

在 WSL 的工作目錄中輸入 `code .`，即可直接使用 Windows 上的 VSCode 開啟該目錄，進行程式碼修改與除錯，體驗與 Windows 原生開發一致。

VSCode 中可直接進行 Git 操作：初始化倉庫、提交程式碼、發佈到 GitHub。也可讓 AI Agent 代勞 Git 操作——讓 Agent 讀取 Windows 宿主機的 Git 設定後同步到 WSL 中，再由 Agent 完成修改、提交與推送。

可透過 Windows 檔案總管查看 Linux 檔案：在左下角 Linux 圖示中選擇對應發行版，或在 WSL 中輸入 `explorer.exe .` 直接開啟當前目錄。Linux 上的 Windows 磁碟以掛載卷形式出現在 `/mnt/c`、`/mnt/d`。注意：專案資料夾盡量放在 Linux 原生目錄，不要放在 Windows 目錄，因為跨作業系統讀取檔案會產生額外 I/O 開銷。

Docker 與 Redis 實戰

17:10

WSL2 最新發行版已預設啟用 systemd，在 WSL 上運行 Docker 的體驗與真正 Linux 電腦一致。從 Docker 官方部署指南複製第 1 和第 4 個命令到 WSL 執行即可完成安裝。

WSL 網路模式與配置檔詳解

24:16

WSL 預設採用 NAT 網路模式，Linux 子系統位於宿主機內部的虛擬網路中，擁有獨立虛擬 IP。透過 localhost 端口自動轉發，宿主機可用 localhost 訪問 WSL 中的服務。但區域網路內的其他設備無法訪問 WSL 中的服務。

鏡像網路模式：在 `C:\Users\<使用者名稱>\.wslconfig` 中新增 `[wsl2]\nnetworkingMode=mirrored`，將 Windows 網卡直接鏡像一份給 WSL 使用，讓兩者處於相同網路環境、使用相同區域網路 IP。還需以管理員身份執行 PowerShell 命令修改防火牆設定。切換後區域網路設備（如手機）即可訪問 WSL 中的服務。

兩個配置檔的區別：`.wslconfig`（Windows 檔案，全域通用，位於 C 槽使用者目錄）可修改記憶體、核心啟動方式、網路模式等；`wsl.conf`（Linux 檔案，位於 `/etc/wsl.conf`，只對單個虛擬機生效）可修改掛載卷、systemd、預設使用者等。示範關閉自動掛載以提升安全性——AI 操作時無法看到 Windows 上的檔案。

進階功能：備份克隆與命令列互通

29:16

備份與克隆：`wsl --export <虛擬機名稱> <壓縮包.tar>` 可將整個 Linux 虛擬機備份成 tar 壓縮包。`wsl --import <新名稱> <安裝路徑> <壓縮包>` 可還原或克隆多份相同環境。這也支援將虛擬機遷移到其他 Windows 電腦。

跨命令列互通：在 PowerShell 中可混寫 Windows 和 Linux 命令。例如 `Get-ChildItem | wsl grep vdu`——前半段是 Windows 的 `Get-ChildItem`，透過管道傳給 Linux 的 `grep` 進行篩選，兩種系統的命令可以無縫協作。

進階技巧

📋 本節指令一覽

  1. wsl --export Ubuntu ubuntu-backup.tar
  2. wsl --unregister Ubuntu
  3. wsl --import Ubuntu D:\wsl\ubuntu ubuntu-backup.tar
  4. wsl --import Ubuntu2 D:\wsl\ubuntu2 ubuntu-backup.tar
  5. Get-ChildItem | wsl grep <關鍵詞>
